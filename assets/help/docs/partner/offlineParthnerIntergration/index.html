Offline File based Integration with Partners

If a new Partner wants to share files for Integration, the same is conveyed to the admin and the following process is followed.

  • Admin (outside LMS ) set up FTP Server Access for that partner and share the same to the partner.
  • LMS admin creates new partner in the LMS
  • Partner Settings are done for the file shared by partner which are described below in detail
  • Partner Admission Instruction(AI)/ Admission Instruction is defined.
  • A Partner Contract is set up for the Admission Instructions.
  • File Integration is used in the Admissions Process.

Partner Settings

Partner settings depend on the file type (i.e.) Inbound and Outbound, also each setting contains different key values explained below.

  • ● Process File Type - It defines which file types will be processed for a particular partner. File can be of following types.
    • Whitelisting File (WL)
    • Blacklisting File (BL)
    • Outbound File (OUT)

    • This setting is named Process_File_type and under this we set key1 to declare the file types.

    Setting Name Key1 Key2 Key3 Key4 Key5
    Process_File_type WL,BL, OUT

    How to set Keys?

    Key1 will be set with the comma separated abbreviations for file type, like if for a particular partner integration only whitelisting file and outbound file are there, then key1 will be set to WL, OUT.

    Key2, Key3, Key4, Key5 are left blank or set as N/A.

Inbound Partner Settings

  • Parsing Implementation Class - This setting is about implementation details of the given file, which can be of two types
    • Comma Separated Values (CSV) - It is a plain text file that contains a list of data which is separated by either comma or tab space.
    • Fixed Width (Flat File) - - It is a plain text file where each field value is the same width and padded with spaces. It is much easier to read than CSV files but takes up more space than CSV.

    • Setting named Parsing_Impl_Class_{process_file_type} is used to parse implementation class and key1, key2 are set to implement this.

    Setting Name Key1 Key2 Key3 Key4 Key5
    Parsing_Impl_C lass_{process_ file_type} csvImportService / fixedLengthImportS ervice tabBasedSplitter / commaBasedSplitter

    How to set Setting Name?

    Parsing_Impl_Class_{process_file_type} will be Parsing_Impl_Class_WL if the Processing file type is Whitelisting File, similarly Parsing_Impl_Class_BL for blacklisting file.

    How to set Keys?

    Key1 - csvImportService( If the Implementation class is CSV) fixedimportService ( If the Implementation class is Fixed Width)

    Key2 - Is set only if Key1 is csvImportService and in this case Key2 defines whether the data will be comma separated (commaBasedSplitter) or tab separated (tabBasedSplitter) and in case of fixedimportService key2 will be empty. If the file is comma separated CSV then also key2 can be left blank.

    Key3, key4 , key5 left blank or N/A.

  • SFTP Details - SFTP details are provided under this setting as
    • Setting Name Key1 Key2 Key3 Key4 Key5
    SFTPDetail_{proce ss_file_type} sftp_path hostname username encrypted password

    How to set Setting Name?

    Setting name will be processed the same as mentioned in case of Parsing_Impl_Class like SFTPDetail_{process_file_type} is SFTPDetail_WL for whitelisting files and likewise for others.

    How to set Keys?

    Key1 holds SFTP path like for example
    /root_folder/offline/inbound/SBIN_WL/

    This path will point to the folder where the file will be uploaded by the partner.

    key2 holds the hostname,

    key3 holds username

    key4 holds the encrypted password. Encryption is done using DESEDE_ENCRYPTION_SCHEME.

    Key5 left blank

  • SFTP File Type - this setting is about the file format and their encryption/decryption details as
    • Setting Name Key1 Key2 Key3 Key4 Key5
    SFTP_File_type_{ process_file_type} gpg/zip Public_key / private_key encrypted password txt fileextentionexist

    How to set Setting Name?

    Same as for above settings

    How to set Keys?

    Key1 - It describes the extension of the encrypted file that can be either .gpg file or .zip file

    Key2 - It is set with the private key / public key name

    Key3 - It contains the encryption password. Encryption is done using DESEDE_ENCRYPTION_SCHEME.

    Key4 - contains the extension of the file

    Key5 - It is set to either fileextensionexist or fileextentionnotexist. if it is fileextensionexist then the key4 data is taken into consideration else not
    Example - suppose there is a file name ABA_PPL_IN_20220412.txt.gpg, in this case key4 is set as txt and key5 is set as fileextensionexist but in case of file ABBPPRLP-20220411.gpg, key4, key5 are set to either blank or N/A.

  • Data Ingestion Webhook - - A webhook is a way for one application to deliver data to another app in real-time. This setting define Webhook for each card detail which is inserted
    • Setting Name Key1 Key2 Key3 Key4 Key5
    Data_Ingestion_Webhook_ {process_file_type} webhook_url 0

    How to set Setting Name?

    Same as for above settings.

    How to set Keys?

    Set the URL under Key1 and set key2 as 0 because currently the system is not reading data from any webhook.

  • PostData Ingestion Webhook- This setting defines Webhook where the ingested data will show
    • Setting Name Key1 Key2 Key3 Key4 Key5
    Postdata_Ingestion_Web hook_{process_file_type} webhook_url 1

    How to set Setting Name?

    Same as for above settings

    How to set Keys?

    Set the URL under Key1 and set key2 as 1

  • Header - This setting is about header information, if it exists or not and the values passed to it, length of the data size, padding and all.
    • Setting Name Key1 Key2 Key3 Key4 Key5
    IsHeaderAvailable_{pro cess_file_type} yes/no Titles any number left/right space /tab

    How to set Setting Name?

    Same as for above settings

    How to set Keys?

    Key1 - defines whether header will be shown in file or not. It is set to either Yes or No.

    Key2 - Defines the header values separated by comma/tab for csv file type and fixed spaces for fixed length file type
    Example- Sno,IssuingRegion,Region,Airport,Outlet,Client_Type,Client_Name,Account_Nam e,Card_Name,Receipt_Number,Quota_Config,Transaction_Date,Transaction_Tim e,Customer_Name,CardNumber,Service_Location,Service_Item

    Key3 - data size used in case of fixed length (can be any number depending on how many characters are needed to show)

    Key4 - defines whether padding will be provided on the left or right.

    Key5 - defines whether the space will be filled by tabs or spaces.

    Key3, key4, key5 are set only for fixed length file case or tab splitted CSV file and if key1 is set no, key2, key3, key4, key5 will be left blank.

  • Parsing Rule Json - It sets up the rules for data validation. The file gets parsed only if it passes these rules, and these rules may be different for each partner.
    • Setting Name Key1 Key2 Key3 Key4 Key5
    Parsing_Rule_Json _{process_file_type } Parsing_Rule_Json Key_mapping

    How to set Setting Name?

    Same as for above settings

    How to set Keys?

    Key1 is set with json code. Sample is shown below 

          {
        "$schema": "http://json-schema.org/draft-04/schema#",
        "title": "Product",
        "description": "A product from the catalog",
        "type": "object",
        "properties": {
        "STARTDATE": {
        "type": "string",
        "pattern": "^\\d{4}-\\d{2}-\\d{2}$"
        },
        "CARDNUMBER": {
        "type": "string",
        "minLength": 16,
        "maxLength": 16
        },
        "OldCardNumber": {
        "type": "string",
        "minLength": 16,
        "maxLength": 16
        },
        "VALIDTILL": {
        "type": "string",
        "pattern": "^\\d{4}-\\d{2}-\\d{2}$"
        },
        "SQ": {
        "type": "string"
        }
        },
        "required": ["STARTDATE", "CARDNUMBER", "VALIDTILL"]
       }
    All the data under the partner file should validate wrt the properties defined in this Parsing_Rule_Json. Like if data under CARDNUMBER in the file doesn’t have the string data type and minLength=maxLength = 16, data validation will be failed and the file will not get parsed. So, the data types, patterns, datalength and some other properties are defined under this which are mandatory to pass.

    Key2 - This key maps the header name to the database key, like below is the example 

        {
      "CARDHOLDER_NO": "CARDNUMBER",
      "No of Access":"NO_OF_ACCESS"
      }
    Here, CARDNUMBER will be mapped to the CARDHOLDER_NO in database

  • PARTNER_MASK - This setting is all about the masking pattern of the card number for a particular partner
    • Setting Name Key1 Key2 Key3 Key4 Key5
    PARTNER_MASK 1111110000111111

    How to set Setting Name?

    Set key1 as the mask pattern like suppose a partner allows 6F6L pattern then the partner mask will be 1111110000111111 for a 16 digit card number. Key2, key3, key4, key5 are left blank or N/A.

  • ONLY_WHITELISTED_ALLOWED - If this is set as ‘true’ under key1, then the cards that fall under valid period as per the whitelisting file are allowed for admission. If the validity period expires for a particular card as per the ingested whitelisting file and a customer seeks admission on the card, he/she will not be allowed.
    • Setting Name Key1 Key2 Key3 Key4 Key5
    ONLY_WHITELISTED_ALLOWED true

Outbound Settings

  • File Frequency - It defines file regeneration cycle duration and next scheduled date. Setup as below
    • Setting Name Key1 Key2 Key3 Key4 Key5
    File_Frequency_{Proc ess_File_type} 30 / 12 9/8/2022 0:00:00 9/7/2022 0:00:00 1 / 2

    How to set Setting Name?

    File_Frequency_{Process_File_type} will be File_Frequency_OUT for this case.

    How to set Keys?

    Key1 - shows the number of days or months based on the Key4 and basically set as a cycle period. If key4 is set 1 that means Key1 contains the number of days and if key4 is set 2 that means key1 contains number of days.

    Key2 - This key is set with the next scheduled datetime.

    Key3 - This key shows Last Run datetime.

    Key4 - as mentioned under Key1 it signifies that what type of data value is set in key1

    Key5 - Left blank

  • Parsing Implementation Class [Parsing_Impl_Class_{process_file_type}] - This setting is about implementation details of the given file, which can be of two types
    • SFTP Details [SFTPDetail_{process_file_type}] - This is same as explained in Inbound Case
    • SFTP File Type [SFTP_File_type_{process_file_type}]- This is set same as explained in Inbound Case
    • Data Ingestion Webhook [Data_Ingestion_Webhook_{process_file_type}] - This is same as explained in inbound, key2 is set as 0 for outbound case.
    • PostData Ingestion Webhook [Postdata_Ingestion_Webhook_{process_file_type}]- This is same as explained in inbound, key2 is set as 0 for outbound case.
    • Password Protected Data It is set if the file is password protected
    • Setting Name Key1 Key2 Key3 Key4 Key5
    PasswordProtectedData_ {process_file_type} yes/no encrypted password

    How to set Setting Name?

    Same as for above settings

    How to set Keys?

    Key2 is set in case if key1 is set yes. It holds the encrypted password. Encryption is done using DESEDE_ENCRYPTION_SCHEME.

  • Header [IsHeaderAvailable_{process_file_type}] - This is set same as explained in Inbound Case
  • Parsing Rule Json This sets up the rules for data validation. The file gets parsed only if it passes these rules, and these rules may be different for each partner.
    • Setting Name Key1 Key2 Key3 Key4 Key5
    Parsing_Rule_Json _{process_file_type } Key_mapping
    How to set Keys?

    Key2 - This key maps the header name to the database key. Like below is the example

    {"SN":"SNO", "Issuing Region":"IssuingRegion"}

    Here, “Issuing Region” will be mapped to the “IssuingRegion” in database Key1, key3, key4, key5 are left blank
